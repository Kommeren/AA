GSoC Ideas List 2015

More Linear Programing Solvers

Brief explanation:

Currently, we provide an interface for creating, modifying and (re)solving LP instances. As the underlying implementation we use the GLPK (GNU Linear Programming Kit) library. Your work would be to add support for more sophisticated solvers:

  • Gurobi
  • CPLEX
Subsequently, we plan to add support for Integer Programming into our framework. As hard as it sounds this task is relatively easy and we strongly believe, that it's more about coding than implementing complex algorithms this time. Except of having fun you will also learn how to use LP solvers and a lot about discrete optimization. For more information see .

Expected results:

Interface for Integer Programming and support for other solvers, and documentation

Knowledge prerequisite:

C++14, Doxygen

Mentor:

Piotr Wygocki

Guillotine Problem Approximation

Brief explanation:

Recently our area of studies moved to family of multidimensional Bin Packing Problems , especially since we have come across new real world data from industry. It becomes clear that previous researchers focused on different problems, especially when it comes to 2D guillotine problems . Recently, we developed novel approach, that relaxes certain constraints in this problem. We've created several prototypes that are using technique called mixed integer programming . We strongly believe that generic solution could save tons of materials in industry, hence we'd like to develop library solution for this problems. During this task you will have a lot of fun learning discrete optimization.

Expected results:

Library code that implements heuristic/exact algorithm for solving Guillotine Problem.

Knowledge prerequisite:

C, C++, Linear Programming, Deep algorithmic knowledge, Discrete Optimization and a lot of communication.

Mentor:

Piotr Wygocki

Biddings for Python and Java

Brief explanation:

The more people are using our Library the better. We belief that most of the people are not familiar with the newest C++, hence we want to facilitate using it for majority of programmers. This task aims to create interface similar to numpy or scipy, but we want to ensure that this biddings will be created automatically. We will not insist on any particular technology, but we require genuine generic solution and we will put pressure on creation of a comprehensive documentation. This task will require deep knowledge of C++14 and frequent communication.

Expected results:

Interface for using PAAL, in Python and Java.

Knowledge prerequisite:

C, C++, Python, Java

Mentor:

Andrzej Pacuk

Recursive Neural Network

Brief explanation:

Recursive neural networks are recent holy grail of machine learning. Problem with it though, is it's inefficiency. Recently we have come into possession of Intel Xeon Phi and we do believe that implementation on this particular architecture will significantly increase it's speed and efficiency. Our goal is to create small machine learning tool program, that will assist programmers on Kaggle competitions, like Vowpal Wabbit

Expected results:

Programming library for the newest Intel Xeon Phi cards.

Knowledge prerequisite:

C, C++, Linux, OpenMP

Mentor:

Krzysztof Ciebiera
